# 로드맵: ConnectHR의 성장 과정

> 10개의 챕터, 하나의 프로젝트. 각 챕터가 끝나면 ConnectHR이 한 단계 성장합니다.

### 이 책이 맞는 분

- Python 기초 문법은 아는 분 (함수, 클래스, 패키지 설치)
- LLM이나 RAG는 처음이거나, 이론만 읽어본 분
- "조각난 예제 말고 하나의 프로젝트를 처음부터 끝까지 만들어보고 싶은" 분

### 한눈에 보기

| 챕터 | 한 줄 요약 | 끝나면 뭐가 보이나 |
|------|-----------|-----------------|
| CH01 | LLM 환각 체험 → RAG 필요성 | 환각 vs RAG 응답 비교 |
| CH02 | 사내 시스템 API 구축 | CRUD API 동작 |
| CH03 | 문서 수집 전략 설계 | 수집 규칙 확립 (개념) |
| CH04 | 파싱→청킹→임베딩→벡터DB | 벡터 검색 + 유사도 |
| CH05 | RAG Q&A 엔진 완성 | 질문 → 답변 + 출처 |
| CH06 | DB+문서 통합 에이전트 | **ConnectHR 탄생** |
| CH07 | 캐시 + 모니터링 | 운영 안정화 |
| CH08 | 검색 품질 튜닝 4가지 | 튜닝 전후 비교 |
| CH09 | 질문 해석 + 근거 시스템 | 쿼리 변환 + 근거 |
| CH10 | 이미지 처리 + 평가 | **성적표 + v1.0 완성** |

---

> **세 가지 전환점**이 이야기를 이끕니다.
>
> 1. **CH01** — LLM 환각을 보고, RAG가 필요하다는 걸 깨닫는다
> 2. **CH06** — RAG만으로는 부족하고, DB와 문서를 합쳐야 한다는 걸 깨닫는다
> 3. **CH08** — 기본 RAG로는 한계가 있고, 튜닝이 필요하다는 걸 깨닫는다
>
> 모든 챕터는 **실패에서 시작**합니다. 에러를 만나고, 원인을 찾고, 해결하는 과정이 곧 학습입니다.

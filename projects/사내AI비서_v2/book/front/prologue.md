# 프롤로그: 이 질문들, 해보신 적 있으시죠?

---

"우리 회사 연차 규정이 어떻게 되나요?"

ChatGPT에 물어봤습니다. 자신 있게 답하더군요. 연차는 15일이고, 3일 전까지 신청하면 된다고. 그런데 우리 회사 규정집을 열어보니 전부 틀렸습니다.

**왜 AI는 그럴듯하게 거짓말을 할까요?** → CH01에서 직접 체험합니다.

---

그래서 우리 회사 문서를 직접 넣어주기로 했습니다. 그런데 질문은 꼬리를 물었습니다.

**"팀원 연차 몇 개 남았어?"는 문서 검색으로 답이 나올까요?** 아닙니다. 이건 데이터베이스에서 꺼내야 합니다. AI 비서가 조회할 사내 시스템부터 만들어야 합니다. → CH02

**공유 드라이브에 PDF, DOCX, XLSX가 뒤섞여 있으면, 전부 넣으면 될까요?** 안 됩니다. 쓰레기를 넣으면 쓰레기가 나옵니다. 어떤 문서를, 어떤 기준으로 골라야 할까요? → CH03

**문서를 골랐으면, 그냥 통째로 넣으면 될까요?** 안 됩니다. 규정집 200페이지를 통째로 넣으면 토큰 한도를 넘깁니다. 문서를 조각내고, 숫자로 바꾸고, 검색 가능한 형태로 저장해야 합니다. → CH04

---

벡터DB에 문서를 넣었습니다. 검색은 됩니다. 그런데 또 질문이 생깁니다.

**검색 결과 다섯 개를 그대로 던져주면 될까요?** "그냥 답을 알려줘"라고 하는 동료에게 검색 결과 목록은 쓸모없습니다. AI가 읽고, 정리하고, 출처까지 붙여서 답해줘야 합니다. → CH05

**"A 사원 연차 몇 개? 그리고 신청 절차는?" — 이 질문에 한 번에 답할 수 있을까요?** "연차 몇 개"는 DB를, "신청 절차"는 문서를 봐야 합니다. 한 질문에 두 가지가 섞여 있으면? → CH06

**같은 질문을 할 때마다 20초씩 기다려야 할까요?** "아까도 물어봤는데 또 기다려?" 캐시, 비용 추적, 에러 처리 — 운영을 안정시키려면 뭐가 필요할까요? → CH07

---

ConnectHR이 동작합니다. 그런데 진짜 문제는 여기서부터입니다.

**"병가 규정을 물어봤는데 왜 출장 규정이 나올까요?"** 검색이 엉뚱한 문서를 가져옵니다. LLM 탓이 아닙니다. 검색 품질을 어떻게 올릴까요? → CH08

**"휴가 규정 알려줘"라고 했는데, 문서에는 "연차유급휴가"라고 적혀 있으면?** 같은 뜻인데 표현이 다르면 못 찾습니다. 질문을 제대로 이해하게 만들려면? → CH09

**PDF 조직도를 넣었는데 "대표이사경영지원본부기술개발본부"가 한 줄로 붙어 나오면?** 이미지 속 정보를 텍스트로 바꾸면 의미가 사라집니다. 그리고 — 이 모든 튜닝이 정말 효과가 있는지, 어떻게 측정할까요? → CH10

---

### 시작하기 전에 — 실습 환경

이 책의 모든 코드는 **로컬에서 무료로** 실행할 수 있도록 설계했습니다.

| 구성 요소 | 요구 사항 | 비고 |
|----------|----------|------|
| Python | 3.12 | 가상환경(venv) 사용 |
| Ollama | 0.5+ | macOS, Linux, WSL2 지원 |
| DeepSeek R1 | 8B 모델 | `ollama pull deepseek-r1:8b` |
| RAM | **16GB 이상** | 8B 모델 로딩 기준 |
| Docker | Compose v2 | PostgreSQL 실행용 (CH02부터) |

**Ollama** 가 DeepSeek R1을 로컬에서 실행합니다. 사내 데이터가 외부 서버로 나가지 않고, API 비용도 없습니다.

> **RAM이 16GB 미만이거나, 로컬 LLM 응답이 너무 느리다면?**
>
> `.env` 파일 한 줄로 OpenAI API로 전환할 수 있습니다.
> ```
> LLM_PROVIDER=openai
> OPENAI_API_KEY=sk-xxxxxx
> ```
> API 비용이 발생하지만, 하드웨어 제약 없이 실습을 진행할 수 있습니다. 각 챕터의 `.env.example` 파일에 설정 예시가 포함되어 있습니다.

---

이 책은 이 질문들의 답을 찾아가는 여정입니다.

환각을 체험하고 → 사내 시스템을 만들고 → 문서를 골라 넣고 → 검색 엔진을 만들고 → 답변을 생성하고 → 에이전트로 통합하고 → 운영을 안정시키고 → 검색을 튜닝하고 → 질문 이해를 높이고 → 성적표로 측정합니다.

하나의 프로젝트, 처음부터 끝까지. 사내 AI 비서 **ConnectHR** 을 함께 만들어봅시다.

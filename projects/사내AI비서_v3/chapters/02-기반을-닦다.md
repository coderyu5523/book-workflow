**팀장**: "오, 괜찮네. 그럼 이것도 돼? '김민준 과장 연차 며칠 남았어?'"

노트북에 입력해봤습니다. RAG가 벡터 DB에서 가장 비슷한 문서를 찾아옵니다. 인사규정이 걸렸습니다. "신입사원은 입사 후 3년 동안 연차가 없다..." LLM이 규정을 읽고 답합니다.

> **LLM**: 김민준 과장은 입사 후 3년 이상 근무하셨으므로 연차가 발생합니다. 정확한 잔여 일수는 확인이 필요합니다.

**팀장**: "확인이 필요하다니. 인사팀 시스템에는 10일 남았다고 나오는데."

(맞다. 규정은 '어떻게 발생하는지'를 알려줄 뿐이다. '지금 며칠 남았는지'는 규정 문서 어디에도 없다.)

### 메뉴판에는 가격이 없다

다시 식당 비유로 돌아가 보겠습니다.

CH01에서 안내원을 고용했습니다. 손님이 "점심 뭐가 있어?"라고 물으면 메뉴판에서 해당 페이지를 찾아서 알려줍니다. 여기까지는 잘 됩니다.

그런데 손님이 이런 질문을 합니다.

"3번 테이블 주문 금액 얼마야?"

안내원이 메뉴판을 아무리 뒤져봐도 이 정보는 없습니다. 메뉴판에는 메뉴 이름과 설명만 있습니다. 3번 테이블이 뭘 주문했는지, 총 금액이 얼마인지는 **주문 장부** 에 기록되어 있습니다.

메뉴판과 주문 장부는 성격이 다릅니다. 메뉴판은 바뀔 일이 거의 없습니다. 한 번 만들어두면 오래 씁니다. 주문 장부는 매 순간 바뀝니다. 새 주문이 들어오면 한 줄이 추가되고, 결제하면 상태가 바뀝니다.

사내 문서도 마찬가지입니다. 취업규칙이나 보안지침은 한 번 만들면 잘 바뀌지 않습니다. RAG로 검색하기에 좋습니다. 그런데 "김민준 과장 연차가 며칠 남았어?"는 다릅니다. 연차를 쓸 때마다 숫자가 바뀝니다. 이런 **정형 데이터** 는 문서가 아니라 데이터베이스에 있어야 합니다.

### 주문 장부를 만들자

안내원이 주문 금액을 답하려면 주문 장부가 먼저 있어야 합니다. 장부가 없으면 아무리 똑똑한 안내원도 답할 수 없습니다.

AI 비서도 같습니다. "연차 며칠 남았어?"에 답하려면 직원 정보와 연차 잔여일을 관리하는 시스템이 먼저 있어야 합니다. 이번 챕터에서는 그 시스템을 만듭니다.

직원 정보, 연차 잔여일, 부서별 매출. 이 세 가지 데이터를 넣고 빼고 고치고 지울 수 있는 **CRUD** 시스템입니다. Create(생성), Read(조회), Update(수정), Delete(삭제)의 앞글자를 딴 이름입니다. 데이터를 다루는 가장 기본적인 네 가지 동작입니다.

주문 장부를 만들고, 그 장부를 열어볼 수 있는 창구까지 함께 만듭니다. 나중에 AI 비서가 이 창구를 통해 데이터를 꺼내갈 겁니다.

(지금은 AI가 아니라 CRUD다. 데이터가 없으면 AI도 답할 수 없으니까.)

## 기술 파트

### 용어 정리

| 이야기 속 표현 | 진짜 용어 | 정식 정의 |
|------------|----------|---------|
| 주문 장부 | 데이터베이스(Database) | 구조화된 데이터를 영구 저장하고 질의할 수 있는 시스템 |
| 장부에 기록하다 | CRUD | Create, Read, Update, Delete. 데이터를 다루는 네 가지 기본 동작 |
| 장부를 열어볼 창구 | REST API | HTTP 메서드(GET, POST, PATCH, DELETE)로 자원을 조작하는 인터페이스 |
| 주문 장부의 양식 | 테이블 스키마(DDL) | 데이터베이스 테이블의 컬럼 구조와 제약 조건을 정의하는 SQL |
| 양식에 맞춘 기록지 | 도메인 모델(dataclass) | 테이블 한 행을 Python 객체로 표현한 클래스 |
| 창구 직원 | FastAPI 라우터 | HTTP 요청을 받아 적절한 함수로 연결하는 웹 프레임워크 구성 요소 |

### 데이터베이스 스키마 --- data/schema.sql

테이블 3개를 만듭니다. 직원, 연차 잔여, 매출. DDL의 핵심 부분만 살펴보겠습니다. 전체 코드는 `code/ex02/data/schema.sql`을 참고하세요.

```sql
CREATE TABLE employee (
    id          SERIAL PRIMARY KEY,
    emp_no      VARCHAR(10)  NOT NULL UNIQUE,
    name        VARCHAR(50)  NOT NULL,
    dept        VARCHAR(50)  NOT NULL,
    position    VARCHAR(50)  NOT NULL,
    hire_date   DATE         NOT NULL
);

CREATE TABLE leave_balance (
    id              SERIAL  PRIMARY KEY,
    employee_id     INTEGER NOT NULL REFERENCES employee(id) ON DELETE CASCADE,
    year            INTEGER NOT NULL,
    total_days      NUMERIC(4,1) NOT NULL,
    used_days       NUMERIC(4,1) NOT NULL DEFAULT 0,
    remaining_days  NUMERIC(4,1) GENERATED ALWAYS AS (total_days - used_days) STORED,
    UNIQUE (employee_id, year)
);

CREATE TABLE sales (
    id          SERIAL PRIMARY KEY,
    dept        VARCHAR(50)  NOT NULL,
    sale_date   DATE         NOT NULL,
    amount      BIGINT       NOT NULL,
    item        VARCHAR(200) NOT NULL
);
```

`employee`는 직원 기본 정보입니다. `leave_balance`의 `remaining_days`는 `total_days - used_days`로 자동 계산되는 컬럼입니다. 연차를 쓸 때마다 `used_days`만 올리면 잔여일이 자동으로 줄어듭니다. `sales`는 부서별 매출 기록입니다.

Docker Compose가 컨테이너를 띄울 때 이 파일을 자동으로 실행합니다. 시드 데이터로 직원 5명, 연차 5건, 매출 10건이 들어갑니다.

### 실습 1 --- app/database.py: DB 연결 관리

아래 코드를 `app/database.py`에 작성합니다.

```python
import os
import sys
from contextlib import contextmanager
from typing import Generator

import psycopg2
import psycopg2.extras
from dotenv import load_dotenv

load_dotenv()

POSTGRES_HOST = os.getenv("POSTGRES_HOST", "localhost")
POSTGRES_PORT = int(os.getenv("POSTGRES_PORT", "5432"))
POSTGRES_DB   = os.getenv("POSTGRES_DB",   "metacoding_db")
POSTGRES_USER = os.getenv("POSTGRES_USER", "metacoding")
POSTGRES_PASSWORD = os.getenv("POSTGRES_PASSWORD", "metacoding_pass")


def get_dsn() -> str:
    return (
        f"host={POSTGRES_HOST} port={POSTGRES_PORT} "
        f"dbname={POSTGRES_DB} user={POSTGRES_USER} "
        f"password={POSTGRES_PASSWORD}"
    )


@contextmanager
def get_connection() -> Generator[psycopg2.extensions.connection, None, None]:
    conn = None
    try:
        conn = psycopg2.connect(
            get_dsn(),
            cursor_factory=psycopg2.extras.RealDictCursor
        )
        yield conn
        conn.commit()
    except psycopg2.OperationalError as exc:
        if conn:
            conn.rollback()
        print(
            "PostgreSQL 서버에 연결할 수 없습니다.\n"
            f"  호스트: {POSTGRES_HOST}:{POSTGRES_PORT}\n"
            "  'docker-compose up -d' 명령으로 DB를 먼저 실행하십시오."
        )
        raise RuntimeError("DB 연결 실패") from exc
    except Exception as exc:
        if conn:
            conn.rollback()
        raise exc
    finally:
        if conn:
            conn.close()


def verify_connection() -> bool:
    try:
        with get_connection() as conn:
            with conn.cursor() as cur:
                cur.execute("SELECT 1")
        return True
    except Exception:
        return False
```

`get_connection()`은 **컨텍스트 매니저** 입니다. `with` 문으로 감싸면 블록이 끝날 때 자동으로 커밋하고 연결을 닫습니다. 에러가 나면 롤백합니다. CRUD 함수마다 연결-커밋-닫기를 반복할 필요가 없습니다.

`RealDictCursor`는 쿼리 결과를 딕셔너리로 돌려줍니다. `row["name"]`처럼 컬럼 이름으로 값을 꺼낼 수 있어서 코드 가독성이 좋아집니다.

`.env` 파일에 DB 접속 정보를 넣어두면 `load_dotenv()`가 환경 변수로 읽어옵니다. 기본값이 `docker-compose.yml`의 설정과 같으므로 로컬 개발 환경에서는 `.env` 없이도 동작합니다.

### 실습 2 --- app/models.py: 도메인 모델

아래 코드를 `app/models.py`에 작성합니다.

```python
from dataclasses import dataclass
from datetime import date
from typing import Optional


@dataclass
class Employee:
    id: int
    emp_no: str
    name: str
    dept: str
    position: str
    hire_date: date


@dataclass
class LeaveBalance:
    id: int
    employee_id: int
    year: int
    total_days: float
    used_days: float
    remaining_days: float
    name: Optional[str] = None
    emp_no: Optional[str] = None


@dataclass
class Sale:
    id: int
    dept: str
    sale_date: date
    amount: int
    item: str


@dataclass
class DeptSummary:
    dept: str

```

`dataclass`는 Python 표준 라이브러리입니다. 테이블의 한 행을 Python 객체로 표현합니다. `Employee` 하나가 직원 테이블의 한 줄에 대응합니다. `LeaveBalance`의 `name`과 `emp_no`는 직원 테이블과 조인할 때 함께 가져오는 필드라서 `Optional`로 선언했습니다.

### CRUD 함수 --- app/crud.py

CRUD 함수는 직원, 연차, 매출 세 도메인에 걸쳐 18개입니다. 패턴이 같으므로 직원 도메인의 `create_employee`와 `get_all_employees` 두 개만 살펴봅니다. 전체 코드는 `code/ex02/app/crud.py`를 참고하세요.

```python
from datetime import date
from typing import Optional
import psycopg2.extensions
from app.models import DeptSummary, Employee, LeaveBalance, Sale


def _generate_emp_no(conn: psycopg2.extensions.connection) -> str:
    sql = "SELECT emp_no FROM employee ORDER BY id DESC LIMIT 1"
    with conn.cursor() as cur:
        cur.execute(sql)
        row = cur.fetchone()
    if row is None:
        return "EMP001"
    last_no = int(row["emp_no"].replace("EMP", ""))
    return f"EMP{last_no + 1:03d}"


def create_employee(
    conn: psycopg2.extensions.connection,
    name: str, dept: str, position: str, hire_date: date,
) -> Employee:
    emp_no = _generate_emp_no(conn)
    sql = """
        INSERT INTO employee (emp_no, name, dept, position, hire_date)
        VALUES (%s, %s, %s, %s, %s)
        RETURNING id, emp_no, name, dept, position, hire_date
    """
    with conn.cursor() as cur:
        cur.execute(sql, (emp_no, name, dept, position, hire_date))
        row = cur.fetchone()
    return _row_to_employee(row)
```

`create_employee`는 사번을 자동 생성합니다. `_generate_emp_no`가 마지막 사번을 조회해서 다음 번호를 만듭니다. `RETURNING` 절 덕분에 INSERT 후 별도 SELECT 없이 생성된 행을 바로 받아옵니다.

```python
def get_all_employees(
    conn: psycopg2.extensions.connection,
    name_filter: Optional[str] = None,
    dept_filter: Optional[str] = None,
) -> list[Employee]:
    conditions: list[str] = []
    params: list[str] = []
    if name_filter:
        conditions.append("name ILIKE %s")
        params.append(f"%{name_filter}%")
    if dept_filter:
        conditions.append("dept ILIKE %s")
        params.append(f"%{dept_filter}%")
    where_clause = ("WHERE " + " AND ".join(conditions)) if conditions else ""
    sql = f"SELECT id, emp_no, name, dept, position, hire_date FROM employee {where_clause} ORDER BY id"
    with conn.cursor() as cur:
        cur.execute(sql, params)
        rows = cur.fetchall()
    return [_row_to_employee(r) for r in rows]


def _row_to_employee(row: dict) -> Employee:
    return Employee(
        id=row["id"], emp_no=row["emp_no"], name=row["name"],
        dept=row["dept"], position=row["position"], hire_date=row["hire_date"],
    )
```

`get_all_employees`는 이름이나 부서로 필터링할 수 있습니다. `ILIKE`는 대소문자를 구분하지 않는 패턴 매칭입니다. 필터가 없으면 전체 직원을 반환합니다.

연차와 매출 CRUD도 같은 패턴입니다. 커넥션을 받아서 SQL을 실행하고 결과를 모델 객체로 변환합니다. 전체 18개 함수의 구조가 동일하므로 나머지는 완성본 레포를 참고하세요.

### API 라우터 --- app/api.py

CRUD 함수를 HTTP로 노출하는 부분입니다. 직원 조회와 등록 엔드포인트만 살펴봅니다. 전체 코드는 `code/ex02/app/api.py`를 참고하세요.

```python
from typing import Optional
from fastapi import APIRouter, HTTPException, Query
from app import crud
from app.database import get_connection
from app.schemas import (
    EmployeeCreate, EmployeeResponse, EmployeeUpdate,
    LeaveBalanceCreate, LeaveBalanceResponse, LeaveBalanceUpdate,
    LeaveUsageRequest, SaleCreate, SaleResponse, DeptSummaryResponse,
)

router = APIRouter(prefix="/api", tags=["API"])

@router.get("/employees", response_model=list[EmployeeResponse], summary="직원 전체 조회")
def api_get_employees(
    name: Optional[str] = Query(None, description="이름 검색 (부분 일치)"),
    dept: Optional[str] = Query(None, description="부서 검색 (부분 일치)"),
) -> list[EmployeeResponse]:
    try:
        with get_connection() as conn:
            employees = crud.get_all_employees(conn, name, dept)
    except RuntimeError as exc:
        raise HTTPException(status_code=503, detail="데이터베이스에 연결할 수 없습니다.") from exc
    return [
        EmployeeResponse(
            id=e.id, emp_no=e.emp_no, name=e.name,
            dept=e.dept, position=e.position, hire_date=e.hire_date,
        )
        for e in employees
    ]
```

`APIRouter`의 `prefix="/api"`가 모든 경로 앞에 `/api`를 붙여줍니다. `get_connection()`으로 DB 연결을 열고 CRUD 함수를 호출합니다. DB 연결 실패 시 503 에러를 반환합니다.

```python
@router.post("/employees", response_model=EmployeeResponse, status_code=201, summary="직원 등록")
def api_create_employee(body: EmployeeCreate) -> EmployeeResponse:
    try:
        with get_connection() as conn:
            emp = crud.create_employee(
                conn, body.name, body.dept, body.position, body.hire_date,
            )
    except RuntimeError as exc:
        raise HTTPException(status_code=503, detail="데이터베이스에 연결할 수 없습니다.") from exc
    return EmployeeResponse(
        id=emp.id, emp_no=emp.emp_no, name=emp.name,
        dept=emp.dept, position=emp.position, hire_date=emp.hire_date,
    )
```

POST 엔드포인트는 `EmployeeCreate` 스키마로 요청 본문을 검증합니다. FastAPI가 JSON을 자동으로 파싱하고 타입을 검증해줍니다. 유효하지 않은 데이터가 들어오면 422 에러를 반환합니다.

### 앱 생성 --- app/main.py

FastAPI 앱을 생성하고 라우터를 등록하는 부분입니다. 핵심 흐름만 살펴봅니다. 전체 코드는 `code/ex02/app/main.py`를 참고하세요.

```python
import os
from dotenv import load_dotenv
from fastapi import FastAPI
from fastapi.responses import RedirectResponse
from fastapi.staticfiles import StaticFiles

load_dotenv()

app = FastAPI(
    title="사내 AI 비서 — CRUD 시스템",
    description="ex02 FastAPI + PostgreSQL CRUD 시스템",
    version="1.0.0",
)

app.mount("/static", StaticFiles(directory="static"), name="static")

from app import views, api
app.include_router(views.router)
app.include_router(api.router)

@app.get("/", include_in_schema=False)
def root() -> RedirectResponse:
    return RedirectResponse(url="/admin/dashboard")
```

`FastAPI()` 인스턴스를 만들고 `include_router()`로 라우터를 등록합니다. `api.router`가 이번 챕터에서 만든 REST API 엔드포인트를, `views.router`가 관리자 웹 UI를 담당합니다. 루트 경로(`/`)에 접속하면 관리자 대시보드로 리다이렉트됩니다.

### 이것만은 기억하자

- "연차 며칠 남았어?"는 문서 검색으로 답할 수 없습니다. 실시간으로 변하는 숫자 데이터는 데이터베이스에 있어야 합니다.
- 이번 챕터에서 만든 CRUD API는 주문 장부와 창구입니다. 데이터를 넣고 빼고 고치고 지울 수 있는 기반 시스템입니다.
- AI 비서는 이 창구를 통해 정형 데이터에 접근하게 됩니다.

숫자 데이터는 준비됐습니다. 그런데 AI 비서가 답해야 할 건 숫자만이 아닙니다. 사내 규정, 보안 지침, 복지 안내처럼 문서에 담긴 정보도 검색할 수 있어야 합니다. 다음 챕터에서는 이 문서들을 어떻게 정리해야 하는지 살펴봅니다.

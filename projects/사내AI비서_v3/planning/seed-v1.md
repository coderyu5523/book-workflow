# 씨앗 — 사내AI비서_v3

## 핵심 메시지

직접 만들어봐야 이해한다. 환각부터 평가까지, 하나의 사내 AI 비서를 처음부터 끝까지 완성하며 RAG의 전체 그림을 익힌다.

## 집필 동기

RAG를 다루는 자료는 많습니다. 그런데 대부분 "벡터DB에 넣고 검색하면 끝"이라는 토막 예제에서 멈춥니다. 환각이 왜 생기는지 직접 겪어보고, 문서를 어떻게 잘라야 하는지 실험하고, 검색 품질을 숫자로 측정하는 데까지 이어지는 자료는 찾기 어렵습니다. 이 책은 "ConnectHR"이라는 하나의 사내 AI 비서를 만들면서, LLM의 한계를 체감하는 첫 번째 코드부터 평가 프레임워크를 돌리는 마지막 코드까지 하나의 흐름으로 관통합니다.

## 타겟 독자

Python 기본 문법은 알지만 LLM과 RAG는 처음 접하는 개발자입니다. API 호출, JSON 파싱 정도는 할 수 있어야 합니다. 반대로 LangChain이나 벡터DB 경험은 전혀 없어도 괜찮습니다.

## 책의 의도

**하려는 것.** 하나의 프로젝트를 처음부터 끝까지 함께 만들며, 각 기술이 "왜" 필요한지 맥락 속에서 전달합니다. 환각을 직접 목격한 뒤에야 RAG를 도입하고, 검색이 엉뚱한 문서를 가져오는 걸 확인한 뒤에야 리랭커를 붙입니다. 문제를 먼저 겪고, 기술을 해결책으로 만나는 구조입니다.

**하지 않으려는 것.** LangChain API 레퍼런스를 나열하지 않습니다. 이론을 설명한 뒤 "직접 해보세요"로 끝내지 않습니다. 모든 선택지를 백과사전처럼 늘어놓지 않습니다.

## 프로젝트 개요

"ConnectHR"은 사내 문서(인사규정 PDF, 매출 보고서 XLSX, 정책 안내 DOCX)를 이해하고 직원 질문에 답하는 AI 비서입니다. 처음에는 LLM에게 질문을 던져 환각을 경험하는 것에서 시작합니다. 사내 데이터를 담은 CRUD API를 만들고, 비정형 문서를 파싱하여 벡터DB에 저장하고, RAG 파이프라인으로 정확한 답변을 생성하고, 정형/비정형 데이터를 통합하는 에이전트로 확장합니다. 그 뒤 운영 안정화, 검색 품질 튜닝, 고급 검색 기법, 평가 프레임워크까지 거치며 실무에 배포할 수 있는 수준으로 성장시킵니다.

## 기술 스택

| 카테고리 | 기술 |
|----------|------|
| 언어/프레임워크 | Python, FastAPI, Jinja2 |
| LLM | Ollama (deepseek-r1:8b, llama3.1:8b), OpenAI API |
| 오케스트레이션 | LangChain 0.3.x (LCEL, ReAct Agent) |
| 벡터DB/검색 | ChromaDB, sentence-transformers (ko-sroberta), BM25, CrossEncoder |
| 정형DB | PostgreSQL 16 (Docker) |
| 문서 파싱 | PyMuPDF, EasyOCR |
| 운영/모니터링 | Langfuse, Rich Console |

## 코드 구성

| 코드 | 내용 |
|------|------|
| ex01 | LLM 환각 체험에서 RAG 필요성 도출 |
| ex02 | FastAPI + PostgreSQL CRUD 시스템 (직원/휴가/매출) |
| ex04 | 문서 파싱(PDF/DOCX/XLSX) + ChromaDB 벡터DB 구축 |
| ex05 | LCEL 기반 RAG Q&A 엔진 + 웹 채팅 UI |
| ex06 | 정형+비정형 통합 ReAct 에이전트 (3단계 라우팅 + MCP 도구) |
| ex07 | 운영 안정화 (재시도/캐시/모니터링/Langfuse) |
| ex08 | 검색 품질 튜닝 — 청킹 전략, 리랭킹, 하이브리드 검색 실험 |
| ex09 | 고급 검색 기법 — 고급 Retriever, 쿼리 재작성, HyDE |
| ex10 | PDF 평가 프레임워크 (OCR/Vision 파싱 + Precision/Recall/MRR/환각률) |

## 비유 전략

**버전업형** 비유를 적용합니다. ex01부터 ex10까지 하나의 시스템 "ConnectHR"을 점진적으로 성장시키는 구조이기 때문입니다. 챕터가 바뀌어도 같은 프로젝트 위에 기능을 얹고, 문제를 해결하고, 품질을 높여갑니다. 따라서 하나의 비유를 챕터마다 확장하는 전략이 자연스럽습니다.

## 차별점

첫째, 환각 체험부터 평가 프레임워크까지 전체 여정을 하나의 프로젝트로 관통합니다. 대부분의 RAG 자료가 "검색 + 생성" 파이프라인에서 끝나는 반면, 이 책은 운영 안정화, 검색 튜닝, 정량 평가까지 다룹니다. 둘째, 모든 기술 도입에 "왜"가 선행합니다. 문제를 먼저 겪고, 그 문제를 해결하기 위해 기술을 도입하는 흐름이므로 각 기술의 존재 이유가 맥락 안에서 드러납니다. 셋째, 로컬 LLM(Ollama)을 기본으로 사용하여 API 비용 부담 없이 실습할 수 있습니다.

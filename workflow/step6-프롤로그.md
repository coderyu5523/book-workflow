## 프롤로그 구조 패턴

### 소절 구조 — 챕터 번호와 일치

소절 번호 **형식**은 `outline.md`의 챕터 번호 형식을 그대로 따른다.
프롤로그는 챕터 0이므로 챕터 번호 앞에 `0.`을 붙인다.

```
outline.md 형식이 Ch.N 이면 → 소절은 Ch.0.N
outline.md 형식이 N 이면    → 소절은 0.N
```

예시 (outline이 Ch.1, Ch.2... 형식일 때):
```
Ch.0.1 현장 문제 발생                ← 반드시 첫 소절 (챕터 이전, 번호 고정)
Ch.0.2 첫 시도와 새 문제 발견        ← Ch.2 내용 대응
Ch.0.3 두 번째 시도와 새 문제 발견   ← Ch.3 내용 대응
...
Ch.0.N 마지막 챕터 완성 + 마무리     ← Ch.N 내용 대응
       ↓
"이 여정에서 만나게 될 것들"          ← 챕터별 한 문단 요약
```

**여러 챕터를 하나의 소절로 묶을 때**: 가장 낮은 챕터 번호를 사용한다.
예) Ch.5와 Ch.6을 묶으면 → `Ch.0.5`

### 소절 0-1 — 현장 문제 발생 (고정 패턴)

- seed.md의 **프롤로그 트리거**를 바탕으로 구체적인 사건으로 시작
- 주인공이 이 기술 없이 겪는 고통을 생생하게 묘사
- 멘토(선배)의 한마디 조언으로 마무리 → 다음 소절의 시작

### 소절 0-2 ~ 0-N — 챕터별 소절 (반복 패턴)

각 소절은 아래 호흡을 따른다:

```
① 배워서 시도해보니 동작함 (짧은 성취감)
② 그런데 새로운 문제 발견 (다음 챕터의 필요성)
③ 멘토에게 질문 → 다음 기술을 자연스럽게 소개
④ 해결 + 짧은 감탄 또는 깨달음
```

마지막 소절은 ②가 없고 "완성"으로 마무리.

### "이 여정에서 만나게 될 것들" 섹션 (고정)

- `planning/outline.md`의 챕터 제목을 그대로 사용한다 — 제목을 새로 만들지 않는다
- 형식: **`Ch.N — [outline.md 챕터 제목]`** + 한 문단 요약
- 챕터에서 **뭘 만드는지, 뭘 배우는지** 위주로 서술
- 마지막 줄: `"자, 그럼 Ch.1에서 만나겠습니다."`

### 캐릭터 규칙

- **주인공**: seed.md의 `저자 이름 / 브랜드명` 필드에서 가져온다 — 없으면 저자에게 확인 후 설정 (예: "메타코딩")
- **멘토**: "선배" 고정 (이름 없음)
- **현장 문제**: seed.md의 `프롤로그 트리거` 필드에서 가져옴

### 금지 규칙

- **코드 블록 없음**: 프롤로그 전체에 코드가 등장하지 않는다
- **기술 정의 없음**: 용어 설명 없이 이름만 자연스럽게 등장 ("Kafka를 써보기로 했습니다" 수준)
- **교과서 어투 금지**: 딱딱한 정의/나열 없이 사건과 대화로만 전달

## 로드맵 작성 규칙

- **시각적**: 버전별 성장을 다이어그램이나 리스트로 보여준다
- **성과 중심**: 각 버전에서 "뭐가 보이는지"를 강조한다
- **한 페이지**: 로드맵은 한 페이지 이내로 간결하게

# Why 분석 규칙

## 5 Whys 절차

1. 실패 현상 확인 (무엇이 실패했는가)
2. 표면적 원인 (왜 실패했는가)
3. 2단계 원인 (왜 그 원인이 발생했는가)
4. 3단계 원인 (왜 그것을 방지하지 못했는가)
5. 근본 원인 (어떤 규칙이 없었기에 이 실수가 가능했는가)

근본 원인에서 **"하지 마라" 규칙 한 줄**을 도출한다.

## 수정 가능한 파일

- `.claude/rules/*.md` — 글로벌 규칙
- `agents/*/AGENT.md ## 규칙` — 에이전트별 규칙
- `agents/editor/AGENT.md 검토 체크리스트` — 검토 체크리스트
- 프로젝트 `review/why-log.md` — 변경 이력 (항상 기록)

## 수정하지 않는 파일

- `CLAUDE.md` — 프로젝트 최상위 설정 (사용자만 수정)
- `agents/*/AGENT.md` — 에이전트 정의 (캐릭터, 스킬 매핑)
- `skills/*/SKILL.md` — 스킬 정의

## 규칙 배치 판단

- 모든 에이전트에 적용되는 규칙 → `.claude/rules/*.md` (글로벌)
- 특정 에이전트에만 해당하는 규칙 → 해당 `agents/*/AGENT.md ## 규칙`

## 규칙 파일 형식

```markdown
## 금지 (Why 추가)
- 이야기 파트에 코드 블록 삽입하지 마라 → why-log.md#2026-03-14-1
- "이것이야말로"로 시작하는 설교 톤 쓰지 마라 → why-log.md#2026-03-14-2
```

## 로그 파일 형식

프로젝트 `review/why-log.md`에 기록.

```markdown
## YYYY-MM-DD-N
- 실패: [어떤 챕터/단계가 실패했는가]
- 원인: [표면적 원인]
- 근본원인: [5 Whys로 파고든 핵심 원인]
- 규칙: [추가된 규칙 경로] "규칙 내용"
```

# Why 스킬 — 같은 실수를 두 번 하지 않는다

## 트리거

1. 편집장 FAIL 판정
2. 빌드 에러 (PDF, 코드)
3. 사용자가 직접 수정 지시
4. 사용자가 "와이", "why", "왜?" 입력

일반 대화/질문/상태확인 → 해당 없음.

## 입출력

| 입력 | 출력 |
|------|------|
| 실패 산출물 + 실패 사유 | 규칙 1줄 추가 + why-log.md 기록 |

## 동작 방식

1. **분석**: 5 Whys 기법으로 근본 원인 추적
2. **규칙 추가**: 해당 파일에 **"하지 마라" 한 줄만** 추가
3. **로그 기록**: 프로젝트 `review/why-log.md`에 상세 이유 + 날짜 + 참조 기록
4. **포인터 연결**: 규칙 옆에 `→ why-log.md#YYYY-MM-DD-N` 참조 표기
5. **재시도**: 업데이트된 규칙으로 다시 실행

## 참조 파일

| 파일 | 내용 |
|------|------|
| `references/why-rules.md` | 5 Whys 절차 + 수정 범위 + 로그 형식 |
